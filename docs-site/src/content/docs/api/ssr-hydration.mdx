---
title: SSR and Hydration
description: Request-scoped SSR extraction, client hydration, critical CSS, and streaming chunk protocol.
---

Use one engine and one SSR renderer per request. This keeps the registry, class names, style order, and extraction output isolated.

```ts
import {
  createSSRRenderer,
  createStyleEngine,
  extractCriticalCSS,
} from 'web-style-engine'

const renderer = createSSRRenderer({ key: 'app', nonce })
const engine = createStyleEngine({ key: 'app', renderer })

const className = engine.css({ color: 'red' }, {
  metadata: { critical: true },
})

const extracted = engine.extract()
```

`styleTags` include a rule id list so the browser renderer can hydrate the existing tag and skip duplicate insertions.

```html
<style data-app="app" data-app-ids="..." nonce="...">...</style>
```

## Client Hydration

```ts
const renderer = createDOMRenderer({ key: 'app' })
renderer.hydrate(document.querySelectorAll('style[data-app="app"]'))

const engine = createStyleEngine({ key: 'app', renderer })
```

If the same rule is generated after hydration, the DOM renderer recognizes the hydrated rule id and reuses the existing style tag.

## Critical CSS

```ts
const critical = extractCriticalCSS(extracted, {
  include: rule => rule.metadata?.critical === true,
})

critical.criticalCss
```

The helper returns the original extraction plus `criticalCss`, so applications can choose whether to inline only critical CSS or the full extracted output.

## Streaming Protocol

`createSSRRenderer()` also exposes `flushChunk()`.

```ts
const firstChunk = renderer.flushChunk()
const secondChunk = renderer.flushChunk()
```

Each chunk contains only rules that have not been flushed before:

```ts
{
  cssText: string,
  rules: StyleRule[],
  styleTags: string,
}
```

This reserves a stable protocol for React, Vue, or Solid streaming integrations without making the core depend on any framework runtime.
