description: "运行时 renderer 实现。"
---

Renderers separate style generation from insertion targets.

Available initial renderers:

- `createDOMRenderer` for browser, Shadow DOM, iframe, extension UI, and micro-frontends
- `createSSRRenderer` for request-scoped extraction, hydration metadata, and streaming chunks
- `createNoopRenderer` for class-name-only or server-only usage
- `createMockRenderer` for unit tests

```ts
createDOMRenderer({
  container: shadowRoot,
  nonce: 'nonce-value',
  insertionPoint: document.querySelector('#style-insertion-point'),
})
```

Hydrate existing SSR style tags before client rendering:

```ts
const renderer = createDOMRenderer({ key: 'app' })
renderer.hydrate(document.querySelectorAll('style[data-app="app"]'))
```

Extract request-scoped SSR output:

```ts
const renderer = createSSRRenderer({ key: 'app', nonce: 'nonce-value' })
const engine = createStyleEngine({ key: 'app', renderer })
const extracted = engine.extract()
```
